# dankoe-writer - Dan Koe 爆款内容生成器

基于 Dan Koe 的 The Modern Writing System 理论，整合多平台调研、爆款框架写作、AI 人性化处理于一体的内容创作工具。

## 核心理念

Dan Koe 爆款内容四步法：
1. **Hook (钩子)** - 3秒抓住注意力（痛点、反常识、好奇心）
2. **Story (故事)** - 用个人经历建立信任和共鸣
3. **Lesson (教训)** - 提炼可复制的方法论和价值
4. **Offer (提议)** - 明确的下一步行动号召

## Usage

```
/dankoe-writer <主题或 niche> [--platforms=平台列表] [--output=目录]
```

**示例**:
- `/dankoe-writer 一人企业如何起步` - 默认搜索所有平台
- `/dankoe-writer AI 副业赚钱 --platforms=reddit,xiaohongshu` - 指定平台
- `/dankoe-writer 远程办公效率 --output=D:/Obsidian/Posts/` - 指定输出目录

## Workflow

```
输入主题
    ↓
[Step 1] 多平台调研
    ├─ Reddit 搜索热门话题和评论
    ├─ 小红书 爆款笔记分析
    ├─ 知乎 高赞问答提取
    ├─ Twitter 金句和观点收集
    └─ (可选) 其他平台补充
    ↓
[Step 2] 素材合成与洞察
    ├─ 痛点汇总（最常被提及的痛苦）
    ├─ 渴望汇总（人们最想要的结果）
    ├─ 常见误区（普遍的错误认知）
    └─ 真实语言（人们实际怎么说话）
    ↓
[Step 3] Dan Koe 框架写作
    ├─ Hook 创作（3秒抓住注意力）
    ├─ Story 构建（个人经历 + 转变）
    ├─ Lesson 提炼（3-5个可复制要点）
    └─ Offer 设计（明确的下一步）
    ↓
[Step 4] AI Humanizer 人性化处理
    ├─ 去除 AI 腔调
    ├─ 加入口语化表达
    ├─ 增加个人化细节
    └─ 调整语气节奏
    ↓
输出多格式内容
    ├─ 长文版 (1500-2000字)
    ├─ 推文/微博线程 (5-10条)
    ├─ 小红书/短平快版 (300-500字)
    └─ 钩子库 (10个备用 Hook)
```

## Execution Steps

### Step 1: 多平台调研

对每个启用的平台，执行以下操作：

#### Reddit 调研
```
使用 WebSearch 或 Browse 搜索：
- site:reddit.com <主题>
- site:reddit.com <主题> frustration
- site:reddit.com <主题> how to
- site:reddit.com <主题> advice

提取：
- 最热帖子的标题和正文
- 高赞评论（特别是带个人故事的）
- 反复出现的痛点词
- 人们常用的缩写和黑话
```

#### 小红书调研
```
搜索关键词：
- <主题> 干货
- <主题> 避坑
- <主题> 我是如何
- <主题> 后悔没早知道

提取：
- 爆款笔记的标题模式
- 开头第一句话
- Emoji 使用习惯
- "姐妹们"、"家人们" 等称呼
- 行动指令（"码住"、"收藏"）
```

#### 知乎调研
```
搜索问题：
- 如何看待 <主题>？
- <主题> 是一种什么样的体验？
- 为什么 <主题> 这么火？
- 有哪些关于 <主题> 的误区？

提取：
- 高赞回答的开头方式
- 专业术语与通俗表达的平衡
- "谢邀"、"先说结论" 等知乎体
- 数据和案例的使用方式
```

#### Twitter/X 调研
```
搜索：
- <主题> (filter:verifed OR min_faves:100)
- "<主题>" "I think"
- "<主题>" "I learned"

提取：
- 金句式表达
- 短平快的观点
- 线程的开头钩子
- 热门标签（Hashtags）
```

#### 其他平台（可选）
- **抖音**: 热门视频标题、评论区神评、BGM 搭配
- **YouTube**: 视频标题模式、Thumbnail 文案、评论区话题
- **微博**: 热搜话题、热门转发、短句表达
- **公众号**: 10W+ 标题、开头段落、互动话术

### Step 3: Dan Koe 框架写作

#### 3.1 Hook 创作（10个备选）

**Hook 类型**:
| 类型 | 公式 | 示例 |
|------|------|------|
| 反常识 | "大多数人都在 X，但其实 Y" | "大多数人都在努力搞副业，但其实90%的副业都赚不到钱" |
| 痛点直击 | "如果你也 X，那 Y" | "如果你也每天加班到10点还看不到希望，那这篇就是写给你的" |
| 数字冲击 | "X 个 Y，改变了 Z" | "3个简单习惯，让我从月薪5k到5万" |
| 个人宣告 | "我 X，然后 Y" | "我辞掉了百万年薪的工作，一点也不后悔" |
| 好奇心 | "X 是一种什么体验？" | "30岁才开始学编程，是一种什么体验？" |

**好 Hook 检查清单**:
- [ ] 前10字能让人停下吗？
- [ ] 制造了认知冲突吗？
- [ ] 目标受众会觉得"这说的就是我"吗？
- [ ] 没有透露全部答案（留悬念）

#### 3.2 Story 构建

**Dan Koe 故事弧线**:
```
1. 过去的我（低谷/挣扎）
   ├─ 具体的痛苦场景
   ├─ 与读者共鸣的细节
   └─ "我曾经也是..."

2. 转折点（发现/遇见）
   ├─ 偶然的发现
   ├─ 关键的顿悟
   └─ "直到有一天..."

3. 转变过程（行动/试验）
   ├─ 具体做了什么
   ├─ 遇到的困难
   └─ 真实的失败经历

4. 现在的我（结果/成果）
   ├─ 量化的成果
   ├─ 生活的改变
   └─ "现在我..."
```

**Story 写作要点**:
- 用具体场景，不要抽象概括
- 加入感官细节（看到、听到、感觉到）
- 暴露脆弱，不要完美人设
- 时间线清晰，有前后对比

#### 3.3 Lesson 提炼

**3-5 个核心要点**:
```
每个要点包含：
1. 核心观点（一句话）
2. 为什么重要（解释价值）
3. 具体怎么做（可操作步骤）
4. 常见错误（提醒避坑）
```

**Lesson 表达方式**:
- "我学到的第一件事是..."
- "这里的关键不是 X，而是 Y"
- "大多数人以为... 实际上..."
- "你需要做的很简单：..."

#### 3.4 Offer 设计

**Offer 类型**:
| 类型 | 适合场景 | 示例 |
|------|----------|------|
| 行动号召 | 短内容 | "今天就去试试 X" |
| 互动邀请 | 社交媒体 | "评论区告诉我，你最想解决的问题是？" |
| 内容引流 | 长文 | "想看更多，关注我，下周更新 Y" |
| 产品引导 | 商业化 | "想系统学习？看我的 Z" |

**好 Offer 特征**:
- 明确具体，不要"请继续关注"
- 低门槛，容易执行
- 与内容主题强相关
- 提供额外价值

### Step 4: AI Humanizer 人性化处理

阅读 `references/humanizer_guide.md` 详细指南，然后执行：

#### 4.1 去除 AI 腔调
- 删除 "综上所述"、"总而言之" 等废话
- 把被动句改成主动句
- 去掉过于正式的表达
- 删除重复啰嗦的部分

#### 4.2 加入口语化表达
- 加入 "说实话"、"你懂吗"、"其实" 等语气词
- 使用短句，避免长难句
- 加入适当的停顿（用换行、破折号）
- 使用第一人称"我"，第二人称"你"

#### 4.3 增加个人化细节
- 加入具体的数字、日期、地点
- 加入个人的小习惯、怪癖
- 加入真实的小错误、小尴尬
- 加入当时的情绪和心理活动

#### 4.4 调整语气节奏
- 快慢结合：短句加速，长句减速
- 制造张力：先提出问题，再给出答案
- 情感变化：从痛苦到希望，从困惑到清晰
- 结尾有力：留下思考或行动

### Step 5: 生成多格式内容

```markdown
[Hook]

[Story - 完整故事弧线]

## Lesson 1: [要点标题]
[内容]

## Lesson 2: [要点标题]
[内容]

## Lesson 3: [要点标题]
[内容]

[总结 + Offer]

---
> 本文由 dankoe-writer 生成
> 基于 Dan Koe 爆款写作框架
```

```markdown
4/[N]: [Lesson 1]

[可选]: 8/[N] - 10/[N]: 扩展内容、互动、引流
```

## Error Handling

| 错误场景 | 处理方式 |
|----------|----------|
| 某个平台无法访问 | 跳过该平台，使用其他平台数据继续 |
| 搜索结果太少 | 扩大关键词范围，或使用通用模板 |
| 内容生成不完整 | 重新生成缺失部分，或提示用户补充 |
| 输出目录不存在 | 创建目录，或使用当前目录 |
